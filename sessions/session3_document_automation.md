## 💪 실습 1: PRD 초안 5분 만에 작성하기 (15분)

### Step 1: 기획 아이디어 준비

다음 기획 아이디어를 사용하거나, 본인의 실제 기획 아이디어를 사용하세요:

```
기능명: 스마트 가격 알림 기능

배경:
- 사용자들이 원하는 숙소 가격이 내려갔을 때 놓치는 경우가 많음
- 경쟁사(야놀자, 에어비앤비)는 이미 가격 알림 기능 제공 중
- 고객 설문 결과, 67%가 가격 알림 기능을 원한다고 응답

핵심 기능:
- 관심 숙소 등록 (찜 기능 활용)
- 가격 하락 시 푸시 알림 발송
- 알림 설정 (10% 이상 하락, 특정 금액 이하 등)
- 알림 히스토리 확인

목표:
- 가격 알림 설정 사용자 수: 월 1만명
- 알림 클릭률: 30% 이상
- 예약 전환율: 알림 클릭 후 5% 이상
```

#### 옵션 A: 간단한 PRD (추천, 워크숍용)

```

   - 현재 문제점

```

   - 무엇을 만드는가?

2. 왜 만드는가? (배경 및 문제)
   - 현재 상황

3. 목표 및 성공 기준
   - 비즈니스 목표

## 🎯 예상 결과물 예시 (일부)

```markdown
# PRD: 스마트 가격 알림 기능

## 📄 문서 정보
- **작성자**: 김민수 (PO)
- **작성일**: 2024년 11월 13일
- **버전**: v1.0
- **상태**: Draft

---

## 1. 개요 (Overview)

### 기능명
스마트 가격 알림 (Smart Price Alert)

### 목적
사용자가 관심 있는 숙소의 가격이 하락했을 때 자동으로 알림을 받아, 예약 기회를 놓치지 않도록 지원하는 기능

---

## 2. 배경 및 문제 정의 (Background)

### 현재 상황
- 앱 사용자의 평균 숙소 탐색 빈도: 주 2.3회
- 즉시 예약하지 않고 가격 비교 후 재방문하는 비율: 68%
- 가격 변동으로 인한 예약 전환 기회 손실 추정: 월 2,000건

### 문제점
1. 사용자가 원하는 가격에 숙소를 예약하지 못함
2. 가격이 내려가도 사용자가 인지하지 못해 기회 손실
3. 앱 재방문율이 경쟁사 대비 낮음 (월평균 8회 vs 경쟁사 12회)

### 시장 분석
| 서비스 | 가격 알림 기능 | 특징 |
|--------|---------------|------|
| 경쟁사 A | ✅ 제공 | 10% 이상 하락 시 알림 |
| 경쟁사 B | ✅ 제공 | 원하는 가격대 설정 가능 |
| 경쟁사 C | ✅ 제공 | 여러 사이트 가격 비교 알림 |
| 우리 서비스 | ❌ 미제공 | - |

---

## 3. 목표 및 성공 지표 (Goals & Metrics)

### 비즈니스 목표
- 사용자 재방문율 증가
- 예약 전환율 향상
- 앱 리텐션 강화

### 핵심 지표 (KPI)

| 지표 | 현재값 | 목표값 | 측정 방법 |
|------|--------|--------|-----------|
| 가격 알림 설정 사용자 수 | 0명 | 월 10,000명 | 알림 설정 완료 수 |
| 알림 클릭률 (CTR) | - | 30% | 알림 클릭 / 알림 발송 |
| 예약 전환율 | - | 5% | 알림 클릭 후 24시간 내 예약 |
| 월평균 재방문율 | 8회 | 10회 | MAU 기준 평균 세션 수 |

---

## 4. 타겟 사용자 (Target Users)

### 주요 페르소나

**페르소나 1: 가격 민감형 여행자**
- **나이**: 25~35세
- **특징**: 최저가를 찾기 위해 여러 플랫폼 비교
- **니즈**: "가격이 떨어질 때 바로 알고 싶어요"

**페르소나 2: 계획형 예약자**
- **나이**: 35~45세
- **특징**: 여행을 미리 계획하고 가격 추이를 관찰
- **니즈**: "원하는 가격대가 되면 자동으로 알려주면 좋겠어요"

### 사용 시나리오

1. **시나리오 A**: 주말 여행 계획
   - 사용자가 부산 해운대 호텔을 검색
   - 마음에 드는 숙소를 찜하고 가격 알림 설정
   - 3일 후 가격이 15% 하락하여 푸시 알림 수신
   - 알림 클릭 후 즉시 예약

2. **시나리오 B**: 명절 숙소 예약
   - 사용자가 설 연휴 제주도 숙소 검색
   - 가격이 비싸서 "30만원 이하로 떨어지면 알림" 설정
   - 2주 후 목표 가격에 도달하여 알림 받음
   - 알림 확인 후 예약 완료

---

## 5. 기능 요구사항 (Requirements)

### 핵심 기능 (Must-have)

#### 5.1 가격 알림 설정
- **설명**: 사용자가 원하는 숙소에 대해 가격 알림을 설정할 수 있음
- **상세**:
  - 찜한 숙소 목록에서 "가격 알림 설정" 버튼 제공
  - 알림 조건 선택:
    - 옵션 1: "가격이 떨어지면 알림" (기본값)
    - 옵션 2: "특정 가격 이하로 떨어지면 알림"
    - 옵션 3: "10% 이상 하락 시 알림"
  - 체크인/체크아웃 날짜 범위 설정
- **예시**: "이 호텔이 20만원 이하가 되면 알려주세요"

#### 5.2 푸시 알림 발송
- **설명**: 설정한 조건에 맞으면 자동으로 푸시 알림 발송
- **상세**:
  - 가격 체크 주기: 매 1시간
  - 알림 내용: "[숙소명] 가격이 [X]% 떨어졌어요! 지금 [Y]원"
  - 알림 클릭 시 해당 숙소 상세 페이지로 이동
  - 하루 최대 알림 횟수 제한: 3회

"기술적 고려사항" 섹션을 더 상세하게 작성해줘.
API 명세, 데이터베이스 스키마, 인프라 요구사항을 포함해줘.
```

### Q3: 여러 기능을 한 번에 PRD로 만들 수 있나요?
**A**: 가능하지만, 각 기능별로 섹션을 분리하는 것을 추천합니다:
```
3개의 관련 기능(A, B, C)을 하나의 PRD로 작성해줘.
각 기능마다 섹션을 분리하되, 공통 배경과 목표는 통합해줘.
```
