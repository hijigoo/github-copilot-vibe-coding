## 📝 지금까지 배운 내용

### ✅ 1. 회의록 자동 정리
- 비정형 텍스트를 구조화된 문서로 변환하는 방법
- 핵심 내용(결정사항, 액션 아이템)을 자동으로 추출하는 기술
- 일관된 형식의 회의록 템플릿 생성

**실무 적용 포인트:**
```
💡 회의 후 5분이면 완벽한 회의록 완성!
   - 회의 중 메모한 내용을 Copilot에 입력
   - 구조화된 회의록 자동 생성
   - 팀원들과 즉시 공유
```

### ✅ 2. 문서 자동 작성
- PRD(Product Requirements Document) 자동 생성
- 사용자 스토리를 상세한 기획서로 확장
- 재사용 가능한 문서 템플릿 제작

**실무 적용 포인트:**
```
💡 기획서 작성 시간 80% 단축!
   - 아이디어나 간단한 요구사항만 입력
   - Copilot이 완성도 높은 PRD 생성
   - 템플릿을 재사용하여 일관성 유지
```

## 🎯 핵심 프롬프트 패턴

이번 섹션에서 배운 효과적인 프롬프트 작성 패턴:

### 1. **구조화 요청**
```
"다음 내용을 [형식]으로 정리해줘"
- 형식을 명확히 지정 (표, 리스트, 섹션 등)
- 포함할 항목을 구체적으로 명시
```

### 2. **컨텍스트 제공**
```
"[역할]로서 [목적]을 위한 문서를 작성해줘"
- 역할: 제품 기획자, PM, 마케터 등
- 목적: PRD 작성, 보고서 작성 등
```

### 3. **예시 기반 생성**
```
"다음 예시를 참고하여 작성해줘"
- 기존 좋은 문서를 참고 자료로 제공
- 원하는 스타일과 톤 명시
```

### 4. **반복 개선**
```
"위 내용에 [요소]를 추가해줘"
- 점진적으로 내용을 보완
- 피드백을 반영하여 개선
```

## 💼 실무 활용 시나리오

### 시나리오 1: 주간 회의록 자동화
```
🔄 매주 반복되는 작업
1. 월요일 팀 미팅 내용 메모
2. Copilot으로 회의록 자동 생성
3. 팀 공유 폴더에 업로드
4. 액션 아이템 자동으로 Jira/Notion에 등록

⏱️ 소요 시간: 30분 → 5분 (83% 감소)
```

### 시나리오 2: 신규 기능 기획서 작성
```
📋 기획서 작성 프로세스
1. 아이디어와 핵심 요구사항 정리 (5분)
2. PRD 템플릿으로 초안 자동 생성 (2분)
3. 세부 내용 보완 및 검토 (15분)
4. 이해관계자 리뷰 요청

⏱️ 소요 시간: 2시간 → 30분 (75% 감소)
```

### 시나리오 3: 고객 인터뷰 정리
```
🎤 인터뷰 후 작업
1. 인터뷰 녹취/메모 준비
2. Copilot으로 핵심 인사이트 추출
3. 사용자 페르소나 문서 자동 생성
4. 개선 아이디어 도출

⏱️ 소요 시간: 1시간 → 15분 (75% 감소)
```

## 💡 마지막 팁

### 문서 자동화 성공의 비결

1. **템플릿화**: 자주 사용하는 문서 형식을 템플릿으로 저장
2. **프롬프트 저장**: 효과적인 프롬프트를 모아서 재사용
3. **점진적 개선**: 한 번에 완벽하지 않아도 OK, 반복하며 개선
4. **검토 필수**: AI가 생성한 내용은 반드시 검토하고 수정

### 실무 적용 체크리스트

- [ ] 팀 회의록 템플릿 만들기
- [ ] PRD 템플릿 커스터마이징하기
- [ ] 자주 쓰는 프롬프트 모음 만들기
- [ ] 팀원들과 프롬프트 공유하기
- [ ] 월간 문서 작성 시간 측정 및 개선
