## 📖 시나리오

> **상황**: 여러분은 주간 제품 회의를 마쳤습니다. 30분간 논의한 내용을 메모했지만, 정리되지 않은 상태입니다. 이제 팀원들에게 공유할 깔끔한 회의록을 만들어야 합니다. 특히 **누가, 언제까지, 무엇을** 해야 하는지 명확히 정리해야 합니다.

**문제점**:
- 회의록 정리에 30분 이상 소요
- Action Item을 빠뜨리는 경우 발생
- 담당자와 마감일이 불명확

**목표**:
- Copilot으로 5분 내 정리 완료
- 모든 Action Item 자동 추출
- 명확한 표 형식으로 정리

## 💪 실습 1: Action Item 자동 추출하기 (10분)

### Step 1: 새 파일 만들기
1. VS Code에서 `⌘ + N` (macOS) / `Ctrl + N` (Windows)
2. `⌘ + S`로 저장: `meeting_action_items.md`

### Step 2: Copilot Chat 열기
```
⌘ + Shift + I (macOS) / Ctrl + Shift + I (Windows)
```

### Step 3: 프롬프트 작성 및 실행

Chat 창에 다음 프롬프트를 입력하세요:

```
위의 회의록에서 다음 정보를 추출해서 Markdown 표로 만들어줘:

1. 액션 아이템 (구체적인 작업 내용)
2. 담당자
3. 마감일 또는 예상 완료일
4. 우선순위 (높음/중간/낮음)

표는 다음 컬럼으로 구성:
| 우선순위 | 작업 내용 | 담당자 | 마감일 | 상태 |

상태는 모두 "진행 전"으로 설정해줘.
```

### Step 4: 결과 검토 및 복사
- Copilot이 생성한 표를 확인
- 필요시 수정 요청 (예: "마감일을 좀 더 구체적으로 해줘")
- 만족스러우면 "Copy" 버튼 클릭

### Step 5: 파일에 붙여넣고 저장
- `meeting_action_items.md`에 붙여넣기
- `⌘ + S` / `Ctrl + S`로 저장

```markdown
| 낮음 | 옵션 필터 UI 개선 (토글 방식) | 이지혜, 최수진 | 다음 스프린트 | 진행 전 |

## ✅ 주요 결정 사항

1. **우선순위 설정**:

   - 3순위: 필터 개선 (다음 스프린트 반영)

2. **목표 지표**:
   - 현재 이탈률: 35%
   - 목표 이탈률: 22% (경쟁사 수준)

3. **배포 일정**:
   - 리뷰 섹션 변경: 11월 17일 (금)
   - A/B 테스트 시작: 배포 직후
```

## 💪 실습 2: 회의록 전체 정리하기 (10분)

```
## 📅 회의 정보
- 날짜
- 참석자 (역할 포함)

(주요 문제점이나 배경을 간단히 요약)

## 💬 주요 논의 사항
(각 논의 사항을 bullet points로 정리)

## ✅ 결정 사항
(구체적인 결정 내용을 번호 매겨 정리)

## 📋 Action Items
(위에서 만든 표 형식 사용)

## 📊 참고 지표
(회의에서 언급된 수치나 목표)

## 🔄 다음 회의
- 일정
- 논의 예정 주제

모든 내용을 Markdown 형식으로 작성해줘.
```

```
1. 회의 기본 정보 (날짜, 시간, 참석자)

6. 다음 회의 안건
7. 참고 자료 링크

Markdown 형식으로 작성해줘.
```

## 💡 실무 활용 팁

### 1. 회의 중 실시간 활용
- 회의록을 메모장에 간단히 작성
- 회의 끝나자마자 Copilot으로 정리 (5분 소요)
- 정리된 내용을 즉시 참석자에게 공유

### 2. 정기 회의 자동화
- 템플릿을 팀 공유 폴더에 저장
- 매주 같은 형식으로 작성
- 지난 회의록과 비교하며 진행 상황 추적

### 3. 다른 팀과 협업
- 일관된 형식으로 회의록 공유
- Action Item을 Jira/Asana 등 프로젝트 관리 도구로 복사
- 마감일 자동 알림 설정
