# 2. 문서 자동 작성

## ⏰ 소요 시간
**30분**

---

## 📖 시나리오

PO로서 신규 기능을 기획해야 합니다. 개발팀에게 전달할 PRD 문서를 작성해야 하는데,  
처음부터 끝까지 작성하려면 2~3시간이 걸립니다. 게다가 어떤 구조로 작성해야 할지 매번 고민됩니다.

**현재 문제:**
- ❌ PRD 작성에 2-3시간 소요
- ❌ 문서 구조가 매번 달라 일관성 부족
- ❌ 중요한 항목을 빠뜨리는 경우 발생

**Copilot으로 해결:**
- ✅ 5분 내에 PRD 초안 완성
- ✅ 표준화된 문서 구조 확립
- ✅ 작성 후 리뷰에만 집중

---

## 📋 실습 재료

다음 기획 아이디어를 사용하거나, 본인의 실제 기획 아이디어를 사용하세요:

```
기능명: 스마트 가격 알림 기능

배경:
- 사용자들이 원하는 숙소 가격이 내려갔을 때 놓치는 경우가 많음
- 경쟁사(야놀자, 에어비앤비)는 이미 가격 알림 기능 제공 중
- 고객 설문 결과, 67%가 가격 알림 기능을 원한다고 응답

핵심 기능:
- 관심 숙소 등록 (찜 기능 활용)
- 가격 하락 시 푸시 알림 발송
- 알림 설정 (10% 이상 하락, 특정 금액 이하 등)
- 알림 히스토리 확인

목표:
- 가격 알림 설정 사용자 수: 월 1만명
- 알림 클릭률: 30% 이상
- 예약 전환율: 알림 클릭 후 5% 이상
```

### Step 2: Copilot Chat에 프롬프트 입력

**💡 Tip**: 처음에는 간단한 버전으로 시작하세요! 필요하면 나중에 확장할 수 있습니다.

#### 옵션 A: 간단한 PRD (추천, 워크숍용)

```
위의 기능 아이디어를 바탕으로 개발팀에게 전달할 간단한 PRD 문서를 작성해줘.
prd_price_alert.md 파일로 저장해줘.

다음 구조로 작성해줘 (각 섹션은 간결하게):

1. 개요 (2-3줄)
   - 기능명, 작성자, 작성일
   - 무엇을 만드는가?

2. 왜 만드는가? (5줄 이내)
   - 현재 문제점
   - 경쟁사 상황

3. 목표 및 성공 기준 (표 형식)
   - 핵심 지표 3-5개

4. 주요 기능 (불릿 포인트)
   - 필수 기능 5개 이내

5. 다음 단계
   - 개발 일정 개요
```

#### 옵션 B: 상세한 PRD (시간 여유 있을 때)

```
위의 기능 아이디어를 바탕으로 개발팀에게 전달할 상세한 PRD 문서를 작성해줘.
prd_price_alert.md 파일로 저장해줘.

다음 구조로 작성해줘:

1. 개요
   - 기능명, 작성자, 작성일
   - 무엇을 만드는가?

2. 왜 만드는가? (배경 및 문제)
   - 현재 상황
   - 문제점
   - 경쟁사는 어떤가?

3. 목표 및 성공 기준
   - 비즈니스 목표
   - 핵심 지표 (측정 가능하게)

4. 누가 사용하는가? (타겟 사용자)
   - 주요 사용자 페르소나
   - 사용 시나리오 예시

5. 어떤 기능인가? (기능 요구사항)
   - 꼭 필요한 기능 (Must-have)
   - 있으면 좋은 기능 (Nice-to-have)

6. 일정
   - 개발 일정
   - 주요 마일스톤

각 섹션마다 구체적인 내용을 채워서 작성해줘.
Markdown 형식으로 작성하고, 표나 bullet points를 적극 활용해줘.

💡 비개발자가 이해하기 쉽게 작성해줘.
```

---

### Step 4: 결과 검토 및 개선

Copilot이 생성한 PRD를 검토하고, 필요하면 다음과 같이 요청하세요:

**더 구체적으로 만들기**:
```
"기능 요구사항" 섹션을 더 구체적으로 작성해줘.
각 기능마다 상세 설명과 예시를 추가해줘.
```

**표 형식으로 변환**:
```
"목표 및 성공 지표" 섹션을 표 형식으로 바꿔줘.
컬럼: 지표명, 현재값, 목표값, 측정 방법
```

**사용자 플로우 다이어그램 추가**:
```
"사용자 플로우" 섹션에 Mermaid 다이어그램을 추가해줘.
사용자가 가격 알림을 설정하고 알림을 받는 전체 과정을 표현해줘.
```

**HTML 문서로 변환하기**:
```
위 PRD 문서를 보기 좋은 HTML 포맷으로 변환해서
prd_price_alert.html 파일로 저장해줘.

다음 스타일 적용:
- 전문적인 레이아웃 (좌측 목차, 우측 내용)
- 섹션별 색상 구분
- 인쇄 가능한 형식
- 회사 로고 위치 (placeholder)
```

## ✏️ 실습 2: 문체 변환하기

같은 내용을 다른 톤으로 변환해봅시다.

### 경영진 보고용으로 변환

```
위의 PRD를 경영진 보고용 요약본으로 변환해줘.

다음 내용만 포함:
- 기능 개요 (2줄)
- 기대 효과 (비즈니스 임팩트)
- 핵심 지표 및 목표
- 예상 개발 기간 및 리소스

공식적이고 간결한 톤으로 작성해줘.
A4 1페이지 분량으로 요약해줘.
```

---

### 개발팀 킥오프용으로 변환

```
위의 PRD를 개발팀 킥오프 미팅용 문서로 변환해줘.

기술적인 내용을 강조하고, 다음을 포함:
- 기능 요구사항 (상세)
- 예상 기술 스택 및 아키텍처
- API 설계 포인트
- 예상 기술적 난이도 및 리스크
- 개발 단계 제안 (Phase 1, 2, 3)

개발자가 이해하기 쉬운 용어로 작성해줘.
```

---

### 마케팅팀 공유용으로 변환

```
위의 PRD를 마케팅팀 공유용 문서로 변환해줘.

다음 내용을 강조:
- 사용자 혜택 (고객 입장에서)
- 경쟁사 대비 차별점
- 프로모션 아이디어 제안
- 예상 런칭 일정

친근하고 설득력 있는 톤으로 작성해줘.
```

## 📋 실습 3: 재사용 가능한 PRD 템플릿 만들기

이제 어떤 기능이든 적용할 수 있는 범용 템플릿을 만들어봅시다.

### PRD 작성 템플릿 생성하기

```
제품 기획자를 위한 PRD 작성 템플릿을 만들어줘.
prd_template.md 파일로 저장해줘.

다음 조건을 만족해야 해:
1. 모든 섹션에 작성 가이드 주석 포함
2. 각 섹션마다 예시 텍스트 제공 (회색 이탤릭체로 표시)
3. 표, 리스트, 다이어그램 등 다양한 형식 활용
4. 비개발자도 쉽게 작성할 수 있도록 친절한 설명

섹션 구성:
1. 문서 정보
2. 개요
3. 배경 및 문제 정의
4. 목표 및 성공 지표
5. 타겟 사용자
6. 기능 요구사항
7. 사용자 플로우
8. 기술적 고려사항
9. 일정 및 마일스톤
10. 리스크 및 의존성
11. Q&A
12. 부록

Markdown 형식으로 작성해줘.
```

## 🎓 추가 도전 과제

### 도전 1: 사용자 스토리 생성
```
위의 PRD를 바탕으로 애자일 개발팀을 위한 
사용자 스토리(User Story)를 생성해줘.

형식: "As a [역할], I want to [기능], so that [목적]"
우선순위를 함께 표시해줘.
```

### 도전 2: FAQ 섹션 추가
```
이 기능에 대해 개발팀이나 이해관계자가 
자주 물어볼 질문 10가지를 예상하고, 
Q&A 형식으로 답변을 작성해줘.
```

### 도전 3: 경쟁사 분석 표 만들기
```
스마트 가격 알림 기능과 유사한 기능을 제공하는 
경쟁 서비스 5개를 조사하고,
비교 분석 표를 만들어줘.

컬럼: 서비스명, 기능 상세, 장점, 단점, 시사점
```
