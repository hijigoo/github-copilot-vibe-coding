# 📝 문서 자동화

## 🎯 이 섹션에서 배울 내용

GitHub Copilot을 활용하여 반복적인 문서 작성 업무를 자동화하는 방법을 배웁니다. 
회의록 정리부터 제품 기획 문서(PRD) 작성까지, 실무에서 바로 활용할 수 있는 기술들을 익혀봅시다.

---

## 📚 세션 구성

### 1️⃣ 회의록 자동 정리 (20분)

**무엇을 배우나요?**
- 비정형 회의 내용을 구조화된 문서로 변환
- 핵심 논의사항, 결정사항, 액션 아이템 자동 추출
- 회의록 템플릿 자동 생성

**실무 활용 예시:**
- 주간 팀 회의록 정리
- 고객 인터뷰 내용 정리
- 브레인스토밍 세션 요약

**소요 시간:** 20분  
**난이도:** ⭐⭐☆☆☆

---

### 2️⃣ 문서 자동 작성 (25분)

**무엇을 배우나요?**
- PRD(Product Requirements Document) 자동 생성
- 사용자 스토리를 PRD로 변환
- 재사용 가능한 문서 템플릿 만들기

**실무 활용 예시:**
- 신규 기능 기획서 작성
- 프로젝트 제안서 작성
- 기술 스펙 문서 생성

**소요 시간:** 25분  
**난이도:** ⭐⭐⭐☆☆

---

## 💡 이 섹션을 마치면

- ✅ 회의록 작성 시간을 80% 단축할 수 있습니다
- ✅ 일관된 형식의 기획 문서를 빠르게 생성할 수 있습니다
- ✅ 문서 작성 프롬프트 작성 방법을 이해합니다
- ✅ 커스텀 문서 템플릿을 만들 수 있습니다

---

## 🎓 학습 목표

1. **효율성**: 문서 작성 시간을 획기적으로 단축
2. **일관성**: 표준화된 형식의 문서 생성
3. **품질**: AI의 도움으로 더 완성도 높은 문서 작성
4. **재사용성**: 한 번 만든 프롬프트를 계속 활용
