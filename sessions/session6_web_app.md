# Session 6: 제품 아이디어 관리 도구 만들기
## ⏰ 소요 시간: 30분 (2:50 – 3:20)

---

## 🎯 학습 목표

이 세션을 마치면:
- PO 실무에 바로 사용할 수 있는 도구 만들기
- 제품 아이디어, 피드백, 요구사항을 구조적으로 관리
- TXT 파일로 저장/불러오기 기능 체험

---

## 📖 왜 이런 도구가 필요한가요?

**PO의 고민**:
- 제품 아이디어가 산재되어 있음 (슬랙, 이메일, 메모장...)
- 고객 피드백을 체계적으로 관리하기 어려움
- 우선순위 판단이 어려움
- 팀원들과 공유가 불편함

**이 도구로 해결**:
- 모든 아이디어를 한 곳에서 관리
- 카테고리별 분류 (아이디어/피드백/요구사항)
- 우선순위 설정 (높음/중간/낮음)
- TXT 파일로 쉽게 저장/공유

---

## 📚 시나리오

**PO인 여러분의 도구**를 만듭니다:
- 제품 아이디어/피드백/요구사항 입력
- 카테고리, 우선순위, 설명 추가
- TXT 파일로 저장하고 불러오기
- 브라우저에서 언제든 사용

---

## 💪 실습: 제품 관리 도구 만들기 (20분)

### Step 1: Copilot Chat 열기
```
⌘ + Shift + I (macOS) / Ctrl + Shift + I (Windows)
```

### Step 2: Copilot에게 명령하기

```
PO를 위한 제품 아이디어 관리 도구를 만들어줘:

파일명: product_manager.html

기능:
1. 입력 폼:
   - 제목 (필수)
   - 카테고리 선택 (아이디어/피드백/요구사항)
   - 우선순위 선택 (높음/중간/낮음)
   - 상세 설명 (텍스트 영역)
   - 추가 날짜 (자동)

2. 목록 표시:
   - 카테고리별로 구분해서 표시
   - 우선순위에 따라 색상 다르게 (높음=빨강, 중간=주황, 낮음=초록)
   - 각 항목마다 수정/삭제 버튼

3. 파일 저장/불러오기:
   - "저장" 버튼: 모든 데이터를 TXT 파일로 다운로드
   - "불러오기" 버튼: TXT 파일을 선택해서 데이터 로드
   - 파일 형식: JSON 형태로 저장

4. 필터링:
   - 카테고리별 필터 버튼
   - 우선순위별 필터 버튼
   - "전체 보기" 버튼

디자인:
- 깔끔한 카드 스타일
- 보라색 계열 그라데이션 배경
- 반응형 (모바일에서도 예쁘게)

모든 코드를 하나의 HTML 파일에 작성하고,
바탕화면에 저장해줘.
```

**💡 Copilot Agent가 자동으로:**
1. ✅ 완전한 HTML 파일 생성
2. ✅ 파일 저장/불러오기 기능 구현
3. ✅ 바탕화면에 저장
4. ✅ 완료!

**여러분이 할 일**: 결과만 확인하면 됩니다!

---

## ✅ Step 3: 도구 실행하기 (2분)

1. 바탕화면에서 `product_manager.html` 파일 찾기
2. 파일을 **더블클릭**
3. 브라우저에서 도구가 열립니다!

**중요**: 서버 없이 바로 작동합니다!

---

## 🎨 Step 4: 기능 테스트 (8분)

### 테스트 시나리오

**1. 제품 아이디어 추가하기**
```
제목: 다크모드 기능 추가
카테고리: 아이디어
우선순위: 높음
설명: 사용자들이 밤에 사용할 때 눈이 편한 다크모드 필요
```

**2. 고객 피드백 추가하기**
```
제목: 검색 속도 개선 요청
카테고리: 피드백
우선순위: 높음
설명: 고객 5명이 검색이 느리다는 피드백 전달
```

**3. 요구사항 추가하기**
```
제목: 결제 시스템 연동
카테고리: 요구사항
우선순위: 중간
설명: 신용카드 외 카카오페이 결제 추가 필요
```

**4. 필터링 테스트**
- "아이디어" 버튼 클릭 → 아이디어만 표시
- "높음" 버튼 클릭 → 우선순위 높은 것만 표시
- "전체 보기" 클릭 → 모두 표시

**5. 저장/불러오기 테스트**
- "저장" 버튼 클릭 → TXT 파일 다운로드
- 항목 몇 개 삭제
- "불러오기" 버튼 클릭 → 저장한 파일 선택
- 원래대로 복원되는지 확인

---

## � 실무 활용 방법

### 이 도구를 어떻게 쓰나요?

**아침 업무 시작할 때**:
- 어제 받은 피드백 입력
- 회의에서 나온 아이디어 정리
- 우선순위 설정

**주간 회의 전**:
- "저장" 버튼으로 TXT 파일 생성
- 팀원들에게 공유
- 함께 우선순위 논의

**스프린트 기획할 때**:
- "높음" 우선순위 항목 필터링
- 이번 스프린트에 들어갈 항목 선정
- 요구사항 상세 설명 참고

**고객 미팅 후**:
- 피드백을 즉시 입력
- 카테고리 분류
- 다음 액션 아이템 도출

---

## 💡 이 도구의 장점

**장점**:
- 📁 **TXT 파일로 저장**: 어디서든 열 수 있고 공유 쉬움
- 🚫 **서버 불필요**: 브라우저만 있으면 OK
- 🔄 **쉬운 공유**: 파일을 슬랙/이메일로 전송
- 📊 **구조적 관리**: 카테고리와 우선순위로 체계적 정리
- 💾 **백업 쉬움**: TXT 파일을 여러 개 저장 가능

**활용 팁**:
- 프로젝트별로 별도 TXT 파일 관리
- 매주 스프린트 시작 전 저장
- 팀 드라이브에 공유해서 협업

---

## 🔧 문제 해결

**Q: 파일이 브라우저에서 열리지 않아요**  
A: 파일 우클릭 → "연결 프로그램" → Chrome 선택

**Q: 저장한 TXT 파일을 어떻게 여나요?**  
A: 메모장이나 텍스트 편집기로 열면 JSON 형식으로 보입니다. 
   도구에서 "불러오기" 버튼으로 다시 가져오면 됩니다.

**Q: 한글이 깨져 보여요**  
A: VS Code에서 파일 저장 시 UTF-8 인코딩 확인

**Q: 데이터를 잃어버리면 어떻게 하나요?**  
A: 정기적으로 "저장" 버튼을 눌러 TXT 파일로 백업하세요!
