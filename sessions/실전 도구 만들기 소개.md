# 🛠️ 실전 도구 만들기

## 🎯 이 섹션에서 배울 내용

지금까지 배운 문서 자동화와 데이터 분석 기술을 종합하여, 실제로 사용할 수 있는 웹 애플리케이션을 만들어봅니다.
**비개발자도 GitHub Copilot과 함께라면 실전 도구를 만들 수 있습니다!**

---

## 📚 세션 구성

### 1️⃣ 제품 아이디어 관리 도구 만들기 (45분)

**무엇을 만드나요?**
- 제품 아이디어를 수집하고 관리하는 웹 애플리케이션
- 데이터베이스 연동 (JSON 파일 기반)
- 아이디어 등록, 수정, 삭제 기능
- 상태별 필터링 및 우선순위 관리

**핵심 기능:**
```
✨ 아이디어 관리
   - 제목, 설명, 담당자, 상태 입력
   - 우선순위 및 예상 임팩트 기록
   - 진행 상태 추적 (백로그/진행중/완료)

📊 시각화
   - 상태별 통계
   - 우선순위별 분포
   - 담당자별 현황

🔍 검색 & 필터
   - 키워드 검색
   - 상태별 필터링
   - 담당자별 필터링
```

**기술 스택 (Copilot이 자동 생성):**
- HTML + JavaScript (프론트엔드)
- Node.js (백엔드 - 선택사항)
- JSON 파일 (데이터 저장)
- Bootstrap (UI 디자인)

**소요 시간:** 45분  
**난이도:** ⭐⭐⭐⭐☆

---

## 💡 왜 이 도구를 만드나요?

### 실무 문제점
```
😰 현재 PM/기획자의 고민
- 아이디어가 이메일, 슬랙, 노션 등 여기저기 흩어짐
- 어떤 아이디어가 논의 중인지 추적 어려움
- 우선순위가 불명확
- 팀원들과 실시간 공유 어려움
```

### 우리가 만들 솔루션
```
✅ 중앙화된 아이디어 관리
- 모든 아이디어를 한 곳에 모아서 관리
- 상태와 우선순위가 명확
- 팀 전체가 실시간으로 확인 가능
- 필터링으로 원하는 정보 빠르게 찾기
```

## 🎓 학습 목표

### 1. **End-to-End 개발 경험**
- 기획부터 구현까지 전 과정 체험
- 프론트엔드 + 백엔드 통합
- 데이터 저장 및 불러오기

### 2. **Copilot 활용 극대화**
- 복잡한 코드도 프롬프트로 생성
- 단계별 기능 추가
- 버그 수정 및 개선

### 3. **실무 적용 가능한 도구**
- 워크숍 끝나고 바로 사용 가능
- 팀 상황에 맞게 커스터마이징
- 다른 도구로 확장 가능 (Slack 봇, Chrome Extension 등)

```
💬 고객 VOC 트래킹
```
