# 문서 자동화 - 마무리

## 🎉 완료!

비정형 회의 메모를 구조화된 회의록으로 바꾸고, 재사용 가능한 회의록 작성 앱까지 완성했습니다.
이제 회의 직후 5분 안에 Action Item을 공유하고, 템플릿 앱을 열어 문서를 작성한 뒤 즉시 txt 파일로 배포할 수 있습니다.

---

## 📖 어떤 문제를 해결했나요?

**Before**
- 회의록 형식이 매번 달라 복붙·서식 정리에 30분 이상 소요
- 담당자/마감일 누락으로 follow-up이 지연
- 공유용 HTML·txt 파일을 만드는 과정이 수동이라 속도가 나지 않음

**After (Copilot 활용)**
- `meeting_summary.md`로 Action Item, 결정 사항, 통계를 5분 안에 생성
- `meeting_action_items.html`에서 담당자별 현황과 차트를 시각화해 즉시 공유
- `meeting_template_app.html`로 누구나 같은 형식으로 회의록을 작성하고 txt 파일로 저장

---

## 📦 이번에 만든 산출물

- `completed/meeting_summary.md`: Action Item 표와 통계가 포함된 Markdown 회의록
- `completed/meeting_action_items.html`: 파이 차트·타임라인이 있는 시각화 리포트
- `completed/meeting_template_app.html`: 재사용 가능한 회의록 작성 앱 (txt 내보내기 포함)

👉 팀 슬랙/드라이브에 업로드해 반복 업무 시간을 줄이세요.

## 📌 Copilot 활용 포인트

**회의록 자동 정리**
- `data/sample_meeting_notes.txt`에서 회의 정보·Action Item·우선순위를 추출
- “담당자별로 그룹화해줘”, “HTML 리포트로 저장해줘”처럼 자연어로 후속 지시

**회의록 작성 앱**
- 필요한 섹션과 저장 방식을 명확히 프롬프트에 열거
- “작성 가이드 주석 추가”, “저장 버튼 클릭 시 txt 다운로드” 같은 세부 요구 조건 전달

**공통 프롬프트 템플릿**
```
[데이터/요구사항]을 바탕으로, [사용자/팀]이 바로 사용할 수 있는
[문서 형식 또는 앱]을 만들어줘.
필수 섹션: [...]
저장 형식: [...]
```
