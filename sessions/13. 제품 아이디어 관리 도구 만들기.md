# 1. 제품 아이디어 관리 도구 만들기

## ⏰ 소요 시간
**25분**

---

## 📖 시나리오

PO로서 제품 아이디어가 슬랙, 이메일, 메모장 곳곳에 흩어져 있습니다.
고객 피드백도 체계적으로 관리하기 어렵고, 우선순위 판단도 어렵습니다.

**현재 문제:**
- ❌ 아이디어가 산재되어 추적 어려움
- ❌ 피드백 관리가 체계적이지 않음
- ❌ 우선순위가 불명확
- ❌ 팀원 공유가 불편함

**만들 도구로 해결:**
- ✅ 한 곳에서 모든 아이디어 관리
- ✅ 카테고리별 분류 (아이디어/피드백/요구사항)
- ✅ 우선순위 명확화 (높음/중간/낮음)
- ✅ 파일로 저장/공유 가능

**💡 비개발자를 위한 설명:**
이번 세션에서는 Copilot이 웹 애플리케이션을 만들어줍니다!
여러분은 원하는 기능만 설명하면 됩니다.

---

## 🛠️ 실습: 제품 관리 도구 만들기

### Step 1: Copilot Chat 열기
Copilot Chat을 열고 아래와 같이 요청하세요.

### Step 2: Copilot에게 명령하기

```
PO를 위한 제품 아이디어 관리 도구를 만들어줘:

파일명: product_manager.html

기능:
1. 입력 폼:
   - 제목 (필수)
   - 카테고리 선택 (아이디어/피드백/요구사항)
   - 우선순위 선택 (높음/중간/낮음)
   - 상세 설명 (텍스트 영역)
   - 추가 날짜 (자동)

2. 목록 표시:
   - 카테고리별로 구분해서 표시
   - 우선순위에 따라 색상 다르게 (높음=빨강, 중간=주황, 낮음=초록)
   - 각 항목마다 수정/삭제 버튼

3. 파일 저장/불러오기:
   - "저장" 버튼: 모든 데이터를 TXT 파일로 다운로드
   - "불러오기" 버튼: TXT 파일을 선택해서 데이터 로드
   - 파일 형식: JSON 형태로 저장

4. 필터링:
   - 카테고리별 필터 버튼
   - 우선순위별 필터 버튼
   - "전체 보기" 버튼

디자인:
- 깔끔한 카드 스타일
- 보라색 계열 그라데이션 배경
- 반응형 (모바일에서도 예쁘게)

모든 코드를 하나의 HTML 파일에 작성하고,
바탕화면에 저장해줘.
```

**💡 Copilot Agent가 자동으로:**
1. ✅ 완전한 HTML 파일 생성
2. ✅ 파일 저장/불러오기 기능 구현
3. ✅ 바탕화면에 저장
4. ✅ 완료!

**여러분이 할 일**: 결과만 확인하면 됩니다!

---

## ✅ Step 3: 도구 실행하기 (2분)

1. 바탕화면에서 `product_manager.html` 파일 찾기
2. 파일을 **더블클릭**
3. 브라우저에서 도구가 열립니다!

**중요**: 서버 없이 바로 작동합니다!

---

## 🎨 Step 4: 기능 테스트 (8분)

### 테스트 시나리오

**1. 제품 아이디어 추가하기**
```
제목: 다크모드 기능 추가
카테고리: 아이디어
우선순위: 높음
설명: 사용자들이 밤에 사용할 때 눈이 편한 다크모드 필요
```

**2. 고객 피드백 추가하기**
```
제목: 검색 속도 개선 요청
카테고리: 피드백
우선순위: 높음
설명: 고객 5명이 검색이 느리다는 피드백 전달
```

**3. 요구사항 추가하기**
```
제목: 결제 시스템 연동
카테고리: 요구사항
우선순위: 중간
설명: 신용카드 외 카카오페이 결제 추가 필요
```

**4. 필터링 테스트**
- "아이디어" 버튼 클릭 → 아이디어만 표시
- "높음" 버튼 클릭 → 우선순위 높은 것만 표시
- "전체 보기" 클릭 → 모두 표시

**5. 저장/불러오기 테스트**
- "저장" 버튼 클릭 → TXT 파일 다운로드
- 항목 몇 개 삭제
- "불러오기" 버튼 클릭 → 저장한 파일 선택
- 원래대로 복원되는지 확인

## 💼 실무 활용 방법

### 이 도구를 어떻게 쓰나요?

**아침 업무 시작할 때**:
- 어제 받은 피드백 입력
- 회의에서 나온 아이디어 정리
- 우선순위 설정

**주간 회의 전**:
- "저장" 버튼으로 TXT 파일 생성
- 팀원들에게 공유
- 함께 우선순위 논의

**스프린트 기획할 때**:
- "높음" 우선순위 항목 필터링
- 이번 스프린트에 들어갈 항목 선정
- 요구사항 상세 설명 참고

**고객 미팅 후**:
- 피드백을 즉시 입력
- 카테고리 분류
- 다음 액션 아이템 도출

## 📋 실제 사용 시나리오

### 시나리오 1: 스프린트 기획

**상황**: 2주 스프린트 시작 전 백로그 정리

**단계별 사용법**:
1. **지난 스프린트 파일 불러오기**
   - "불러오기" 버튼 클릭 → `sprint_22.txt` 선택
   
2. **완료된 항목 정리**
   - 완료된 항목 삭제
   - 남은 항목은 우선순위 재조정

3. **새 피드백 추가**
   - 지난 2주간 받은 고객 피드백 입력
   - 카테고리 분류 (아이디어/피드백/요구사항)

4. **우선순위 필터링**
   - "높음" 버튼 클릭 → 개발팀과 함께 검토

5. **스프린트 백로그 저장**
   - "저장" → `sprint_23.txt`로 저장
   - 팀 Slack 채널에 공유

**소요 시간**: 15-20분

---

### 시나리오 2: 주간 제품 회의 준비

**상황**: 매주 금요일 제품 회의 준비

**단계별 사용법**:
1. **이번 주 추가 내용 확인**
   - 월-금 입력한 항목 확인 (총 12개)

2. **카테고리별 정리**
   - "아이디어" 버튼 → 신규 기능 검토
   - "피드백" 버튼 → 개선 요청 확인
   - "요구사항" 버튼 → 필수 기능 체크

3. **회의 자료 준비**
   - "전체 보기"로 전환
   - 화면 공유 준비 또는 스크린샷
   - 중요 항목 3개 강조

4. **회의 후 액션**
   - 우선순위 변경 반영
   - "저장" → `weekly_meeting_2024_11_15.txt`

**소요 시간**: 10분

---

### 시나리오 3: 고객 VOC 관리

**상황**: 고객센터로부터 일주일 치 VOC 전달받음

**단계별 사용법**:
1. **VOC 일괄 입력** (25건)
   - 제목, 카테고리, 우선순위, 설명 입력

2. **우선순위별 분류**
   - "높음" (긴급): 7건 → 즉시 개발팀 전달
   - "중간" (중요): 12건 → 다음 스프린트
   - "낮음" (개선): 6건 → 장기 계획

3. **패턴 분석**
   - "결제" 관련: 8건 → 결제 시스템 점검
   - "검색" 관련: 5건 → UX 개선 고려
   - "성능" 관련: 4건 → 서버 최적화

4. **월간 리포트용 저장**
   - "저장" → `voc_november_week2.txt`

**소요 시간**: 40분

### 시나리오 4: 분기 로드맵 수립

**상황**: Q1 제품 로드맵 작성

**단계별 사용법**:
1. **지난 분기 파일 통합**
   - `q4_ideas.txt` 불러오기
   - 완료되지 않은 항목 확인

2. **신규 아이디어 추가**
   - 경영진 요청사항

   - 경쟁사 벤치마킹 내용

3. **우선순위 재배치**

   - 기술적 난이도 평가

   - "높음": Q1 필수 (3개월 내)
   - "중간": Q2 검토 (4-6개월)

**활용 팁**:
- 프로젝트별로 별도 TXT 파일 관리
- 매주 스프린트 시작 전 저장

## 🔧 문제 해결

**Q: 파일이 브라우저에서 열리지 않아요**  
A: 파일 우클릭 → "연결 프로그램" → Chrome 선택

**Q: 저장한 TXT 파일을 어떻게 여나요?**  
A: 메모장이나 텍스트 편집기로 열면 JSON 형식으로 보입니다. 
   도구에서 "불러오기" 버튼으로 다시 가져오면 됩니다.

**Q: 한글이 깨져 보여요**  
A: VS Code에서 파일 저장 시 UTF-8 인코딩 확인

**Q: 데이터를 잃어버리면 어떻게 하나요?**  
A: 정기적으로 "저장" 버튼을 눌러 TXT 파일로 백업하세요!
