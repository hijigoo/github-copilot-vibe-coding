# Session 4: UX 리서치 분석
## ⏰ 소요 시간: 30분 (1:50 – 2:20)

---

## 🎯 학습 목표

이 세션을 마치면:
- 10개 이상의 사용자 피드백을 자동으로 분류하고 분석
- 공통 키워드 및 불만사항 자동 도출
- 페르소나별 피드백 그룹핑

## 📖 시나리오

> **상황**: 여러분은 UX 리서처입니다. 최근 신규 숙박 검색 기능에 대해 15명의 사용자를 인터뷰했습니다. 각 인터뷰 내용을 정리했지만, 이제 이 데이터를 분석하여 공통 패턴을 찾고 인사이트를 도출해야 합니다.

**문제점**:
- 피드백 데이터가 많아 분석에 시간 소요
- 수작업으로 분류하다 보면 주관적 판단 개입
- 중요한 인사이트를 놓칠 가능성

**목표**:
- Copilot으로 객관적인 분석 수행
- 페르소나별 니즈 자동 분류
- 우선순위 높은 이슈 식별

## 💪 실습 1: 피드백 자동 분류 및 키워드 추출 (10분)

### 🎯 목표
Copilot에게 명령만 하면 **자동으로 분석되고 결과가 나옵니다**!

### Step 1: Copilot Chat 열기
```
⌘ + Shift + I (macOS) / Ctrl + Shift + I (Windows)
```

### Step 2: Copilot에게 명령하기

Chat 창에 다음과 같이 요청하세요:

```
sessions/data/user_feedback_data.txt 파일을 분석해줘:

1. 가장 많이 언급된 불만사항 Top 5
2. 각 불만사항별 언급 횟수
3. 불만사항을 카테고리별로 분류 (성능, UI/UX, 기능, 가격, 콘텐츠)
4. 각 카테고리별 주요 키워드
5. 페르소나별 피드백 그룹화 (연령대, 직업)

결과를 Markdown 표 형식으로 보여줘.
```

### Step 3: 강사와 함께 코드 실행하기

**💡 강사가 함께합니다**: Copilot이 코드를 생성하면, 강사가 단계별로 실행 방법을 알려드립니다.

**진행 순서**:
1. Copilot이 Python 코드를 생성합니다
2. 강사가 "이 코드를 여기에 복사하세요" 안내
3. 강사가 "이 버튼을 눌러서 실행하세요" 안내
4. 함께 결과를 확인합니다

**처음 1-2번은 강사가 직접 시연**하고, 이후에는 여러분이 따라해봅니다.

### Step 4: 추가 분석 요청

만족스럽지 않으면 계속 요청:

```
감정 분석도 추가해줘 (긍정/부정/중립)
```

```
심각도를 높음/중간/낮음으로 자동 판단해줘
```

```
개선 우선순위를 1~10점으로 계산해줘
```

**HTML 리포트로 변환하기**:
```
위 UX 리서치 분석 결과를 인터랙티브한 HTML 리포트로 만들어서
ux_research_report.html 파일로 저장해줘.

다음 내용 포함:
- 불만사항 Top 5 (차트)
- 카테고리별 분석 (접을 수 있는 섹션)
- 페르소나별 인사이트 (카드 레이아웃)
- 개선 로드맵 타임라인
- 반응형 디자인 (모바일 지원)
- 인쇄 가능한 스타일
```

### Step 4: 추가 분석 요청

**심각도 평가 요청**:
```
위의 불만사항들을 심각도 (높음/중간/낮음)로 분류하고,
각 불만사항이 비즈니스에 미치는 영향을 평가해줘.

표 형식으로 정리:

**페르소나별 분류 요청**:
```

각 페르소나별로:
- 주요 특징
- 핵심 니즈
- 주요 불만사항

표 형식으로 정리해줘.
```

**HTML 대시보드로 변환하기**:
```

ux_analysis_dashboard.html 파일로 저장해줘.

다음 기능 포함:
- 불만사항 Top 5 차트 (Chart.js)
- 카테고리별 파이 차트
- 페르소나별 탭 구조

## 🏷️ 카테고리별 상세 분석

### 1. 성능 이슈 (5건)
**주요 키워드**: 느림, 로딩, 대기 시간, 3초, 5초

**구체적 불만**:
- 자동완성 속도 (3초 대기)
- 검색 결과 로딩 속도
- 이미지 로딩 속도
- 필터 적용 시 딜레이 (5초 이상)

**비즈니스 영향**: 
- 이탈률 증가 (현재 35%)
- 경쟁사 대비 열위 (야놀자 언급)
- 사용자 만족도 하락

**개선 우선순위**: 🔴 **최우선**

---

### 2. UI/UX 이슈 (4건)
**주요 키워드**: 불편, 찾기 어려움, 직관적이지 않음, 구식

**구체적 불만**:
- 필터 버튼 위치 (모바일)
- 정렬 기준 불명확
- 날짜 선택 인터페이스
- 전체적인 디자인 구식

**비즈니스 영향**:
- 사용성 저하
- 브랜드 이미지 손상
- 첫인상 부정적

**개선 우선순위**: 🟠 **높음**

---

### 3. 콘텐츠/신뢰도 이슈 (3건)
**주요 키워드**: 리뷰, 사진, 신뢰, 실제와 다름

**구체적 불만**:
- 리뷰 신뢰도 낮음
- 사진 리뷰 부족
- 숙소 사진과 실제 차이

**비즈니스 영향**:
- 예약 전환율 저하
- 취소율 증가 가능성
- 재방문 의사 감소

**개선 우선순위**: 🟡 **중간**

- 빠른 예약
- 영수증/세금계산서
- 검색 조건 저장

**주요 불만사항**:
- 영수증 발급 불편
- 검색 조건 저장 기능 없음
- 비즈니스 호텔 필터 부족

**제안**:
- 비즈니스 사용자 전용 기능 추가
- 자동 세금계산서 발급

## 💡 핵심 인사이트

### 1. 성능이 가장 큰 문제
- **33%의 사용자**가 속도 관련 불만 제기
- 경쟁사 대비 명확한 열위 인식
- 🎯 **액션**: 검색 속도 개선을 최우선 과제로 설정

### 2. 모바일 경험 개선 시급
- 모바일 UI/UX 관련 불만 다수
- 특히 필터, 날짜 선택 등 핵심 기능에서 문제
- 🎯 **액션**: 모바일 UX 리뉴얼 프로젝트 착수

### 3. 콘텐츠 신뢰도 강화 필요
- 리뷰와 사진의 신뢰성 문제
- 사용자 생성 콘텐츠 부족
- 🎯 **액션**: 사진 리뷰 인센티브 프로그램 도입

### 4. 페르소나별 맞춤 기능 필요
- 연령대/목적별 니즈 차이 명확
- 비즈니스 사용자 전용 기능 부재
- 🎯 **액션**: 페르소나 기반 기능 로드맵 수립
