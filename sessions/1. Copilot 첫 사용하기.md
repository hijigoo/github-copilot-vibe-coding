# 2. GitHub Copilot 첫 사용하기

## ⏰ 소요 시간
**10분**

---

## 📖 시나리오

이제 환경 설정이 완료되었으니 GitHub Copilot을 직접 사용해봅시다!
마치 똑똑한 동료와 대화하듯이 Copilot과 소통하는 방법을 배웁니다.

**이 세션의 목표:**
- Copilot Chat 사용법 익히기
- 첫 프롬프트 작성 경험
- 효과적인 프롬프트 패턴 학습

---

## 💬 실습 1: 첫 대화 시작하기

### Step 1: Copilot Chat 열기

1. 키보드에서 `Cmd + I` (Mac) 또는 `Ctrl + I` (Windows) 누르기
2. 화면에 Chat 창이 나타나면 성공!

**💡 Tip**: 왼쪽 사이드바에서 채팅 아이콘을 클릭해도 됩니다.

### Step 2: 간단한 인사

**프롬프트**:
```
안녕하세요! 저는 비개발자입니다. GitHub Copilot이 무엇을 도와줄 수 있나요?
```

**기대 결과**:
- Copilot이 자신의 기능을 설명합니다
- 비개발자가 활용할 수 있는 방법을 제안합니다

---

## 📊 실습 2: 간단한 문서 생성하기

### Step 1: 문서 생성 요청

**프롬프트**:
```
회의 참석자 5명의 이름과 역할을 표로 만들어줘. 
참석자는 김철수(PO), 이영희(UX), 박지민(마케터), 최민수(개발자), 정수진(디자이너)야.
```

**기대 결과**:
Copilot이 다음과 같은 표를 생성합니다:

| 이름 | 역할 |
|------|------|
| 김철수 | PO |
| 이영희 | UX |
| 박지민 | 마케터 |
| 최민수 | 개발자 |
| 정수진 | 디자이너 |

---

## ✍️ 실습 3: 좋은 프롬프트 작성법 익히기

### 핵심 원칙

**구체적으로 말하기**
```
❌ "보고서 만들어줘"
✅ "지난주 프로젝트 진행 상황 보고서를 만들어줘. 
    완료된 작업, 진행 중인 작업, 다음 주 계획으로 구성해줘."
```

**형식 지정하기**
```
❌ "데이터 정리해줘"
✅ "이 데이터를 표 형식으로 정리해줘."
```

**핵심**: 명확하게 말할수록 원하는 결과가 나옵니다.

---

## 🎯 예상 결과물

이 세션을 완료하면:
- ✅ Copilot Chat을 자유롭게 열고 사용
- ✅ 간단한 표와 문서를 생성할 수 있음
- ✅ 좋은 프롬프트 vs 나쁜 프롬프트를 구분
- ✅ 명확하고 구체적인 요청 작성

**확인 방법:**
```
Copilot Chat에서 다음 프롬프트 테스트:
"오늘 배운 내용을 bullet points 3개로 요약해줘"
```

---

## 🎓 추가 도전 과제

기본 사용법을 익혔다면 다음을 시도해보세요:

### 도전 1: 다양한 형식으로 요청하기
```
"회의 참석자 5명의 정보를 다음 형식으로 만들어줘:
1. 마크다운 표
2. bullet points 리스트
3. JSON 형식"
```

### 도전 2: 단계별 설명 요청
```
"비개발자가 GitHub Copilot으로 업무 자동화를 시작하는 
3단계 가이드를 만들어줘. 각 단계마다 구체적인 예시를 포함해줘."
```

### 도전 3: 프롬프트 개선하기
```
나쁜 프롬프트: "보고서 만들어줘"
→ 어떻게 개선하면 좋을지 Copilot에게 물어보세요:
"이 프롬프트를 더 구체적으로 개선해줘"
```
