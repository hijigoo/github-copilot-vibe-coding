## ⏰ 소요 시간: 15분 (2:05 – 2:20)

## 🎯 학습 목표

이 세션을 마치면:
- 정성 데이터와 정량 데이터를 통합 분석
- 데이터 기반 개선 우선순위 수립
- 인사이트를 실행 가능한 액션 아이템으로 변환
- 종합 분석 리포트 생성

## 💪 실습 1: 정량 데이터 추가 분석 (7분)

실제 업무에서는 정성 데이터(인터뷰)와 정량 데이터(통계)를 함께 분석합니다.

### 샘플 데이터 추가

앞 세션에서 분석한 인터뷰 데이터에 다음 통계 데이터를 추가로 제공합니다:

```text
=== 앱 사용 통계 데이터 (최근 30일) ===

검색 기능 사용 현황:
- 전체 검색 수: 1,250,000회
- 자동완성 사용률: 78%
- 필터 사용률: 45%
- 평균 검색 시간: 3.8초
- 결과 페이지 이탈률: 35%

페이지별 체류 시간:
- 검색 결과 페이지: 평균 28초
- 숙소 상세 페이지: 평균 1분 42초
- 예약 페이지: 평균 3분 15초

클릭 히트맵 (모바일):
- 검색창: 95%
- 필터 버튼: 12% (찾기 어려움)
- 정렬 옵션: 8%
- 지도 보기 버튼 (없음): N/A

사용자 이탈 지점:
- 검색 결과 대기 중: 25%
- 필터 설정 중: 15%
- 숙소 상세 페이지: 10%
- 예약 페이지: 5%
```

### 프롬프트

Copilot Chat에 다음과 같이 요청하세요:

```
이전에 분석한 UX 인터뷰 데이터와 위의 통계 데이터를 결합하여 종합 분석해줘.

다음 내용 포함:
1. 정성/정량 데이터가 일치하는 부분 (트라이앵귤레이션)
   - 인터뷰에서 나온 불만이 실제 수치로도 확인되는 항목
   
2. 데이터 간 불일치 사항 및 추가 조사 필요 항목
   - 인터뷰에서 언급되지 않았지만 수치상 문제인 부분
   - 인터뷰에서 언급되었지만 수치상 큰 문제 아닌 부분

3. 데이터 기반 개선 우선순위
   - 정성+정량 데이터 모두에서 문제인 항목 (최우선)
   - 정량 데이터만 문제인 항목 (높음)
   - 정성 데이터만 문제인 항목 (중간)

4. 예상 개선 효과 (정량적 목표)
   - 각 개선사항이 적용되면 예상되는 수치 변화

표와 bullet points를 활용해서 보기 쉽게 정리해줘.
```

### 예상 결과물 예시

```markdown
# UX 리서치 종합 분석 리포트

## 📊 데이터 트라이앵귤레이션 (일치 항목)

| 문제점 | 인터뷰 증거 | 통계 증거 | 심각도 |
|--------|------------|-----------|--------|
| 검색 속도 느림 | 5명 언급 (33%) | 평균 3.8초, 이탈률 25% | 🔴 최우선 |
| 필터 UI 불편 | 4명 언급 (27%) | 클릭률 12% (매우 낮음) | 🔴 최우선 |
| 정렬 기준 불명확 | 3명 언급 (20%) | 정렬 옵션 클릭 8% | 🟠 높음 |

**인사이트**: 
- 정성/정량 데이터 모두에서 검증된 3가지 문제는 **즉시 해결 필요**
- 특히 필터 클릭률 12%는 치명적 (업계 평균 40~50%)
- 검색 속도로 인한 이탈률 25%는 매출 직결

---

## ⚠️ 데이터 불일치 항목

### 1. 통계로만 발견된 문제
| 문제점 | 통계 증거 | 인터뷰 언급 | 분석 |
|--------|-----------|------------|------|
| 예약 페이지 이탈 | 5% | 없음 | 사용자가 인지하지 못하는 숨은 문제 |

**추가 조사 필요**: 예약 페이지에서 이탈하는 5%의 사용자 심층 인터뷰

### 2. 인터뷰로만 발견된 문제
| 문제점 | 인터뷰 증거 | 통계 증거 | 분석 |
|--------|------------|-----------|------|
| 모바일 디자인 구식 | 3명 언급 | 체류시간/이탈률 정상 | 주관적 불만, 기능적 문제는 아님 |

**우선순위 조정**: 디자인 개선은 기능 개선 후 진행

| 4 | 리뷰 신뢰도 강화 | 정성 20% + 전환율 관련 | 전환율 +10% |

### P2 (중간 - 3개월 이후)
| 순위 | 개선 항목 | 근거 | 예상 효과 |

| 6 | 디자인 리뉴얼 | 정성 주관적 | 브랜드 이미지 |
```

## 💪 실습 2: 인사이트를 액션 아이템으로 변환 (8분)

분석 결과를 실행 가능한 작업으로 변환해봅시다.

### 프롬프트

```
위의 UX 리서치 종합 분석을 바탕으로 
실행 가능한 액션 아이템 리스트를 만들어줘.

각 액션 아이템마다 다음 정보 포함:
- 작업 내용 (구체적으로)
- 담당 팀 (UX/개발/마케팅/데이터 등)
- 예상 소요 기간 (일 단위)
- 예상 난이도 (상/중/하)
- 성공 측정 지표 (KPI)
- 기대 효과 (정량적)
- 우선순위 (P0/P1/P2)

표 형식으로 작성하고, 우선순위별로 그룹핑해줘.

추가로:
- 팀 간 의존성 표시 (A팀 완료 후 B팀 시작)
- 타임라인 (간트 차트 스타일)
```

### 예상 결과물 예시

```markdown
# UX 개선 액션 아이템

| 항목 | 내용 |

## 🟠 P1: 높음 (Week 3~8)
```

## 💡 실무 활용 팁

### 1. 리서치 후 즉시 분석
- 인터뷰 직후 Copilot으로 즉시 분석
- 기억이 생생할 때 추가 인사이트 도출

- 인터뷰 + 설문조사 + 앱 분석 데이터

- 월간/분기별 피드백 누적 분석
- 시간에 따른 트렌드 변화 추적
- 개선 효과 측정

### 4. 액션 아이템 추적
- GitHub Issues 또는 Jira와 연동
- 주간 스탠드업에서 진행 상황 공유

```
위의 인사이트를 바탕으로,

각 항목마다 구체적인 예시와 데이터를 포함해줘.
```

### 도전 2: 사용자 여정 지도 생성
```
가장 불만이 많았던 "빠른 예약형" 페르소나의
사용자 여정 지도(User Journey Map)를 만들어줘.

각 단계마다:
- 터치포인트
- 사용자 행동
- 감정 (😊/😐/😞)
- 불만사항
- 개선 기회

Mermaid 다이어그램 또는 표 형식으로 작성해줘.
```

### 도전 3: A/B 테스트 제안서
```
위의 개선 아이템 중 상위 3개에 대해
A/B 테스트 제안서를 작성해줘.

각 테스트마다:
- 가설 (Hypothesis)
- 테스트 버전 (A vs B)
- 측정 지표 (Primary/Secondary)
- 샘플 사이즈 계산
- 예상 소요 기간
- 예상 결과
- 통계적 유의성 판단 기준
```

## 🔍 자주 묻는 질문

### Q1: 피드백이 100개 이상이면 어떻게 하나요?
**A**: 데이터를 카테고리별로 나누어 분석하세요:
```
위의 피드백 중 "성능" 관련된 내용만 추출해서 상세 분석해줘.
```

### Q2: 상반된 의견이 있으면 어떻게 처리하나요?
**A**: Copilot에게 충돌 분석을 요청하세요:
```
위의 피드백 중 서로 상반되는 의견을 찾아서,
각 의견의 비율과 배경(연령대, 사용 패턴)을 분석해줘.
```

### Q3: 수치로 우선순위를 정하고 싶어요
**A**: 스코어링 시스템을 요청하세요:
```
각 불만사항에 대해 다음 기준으로 점수를 매겨줘:
- 영향도 (Impact): 언급 빈도 × 심각도 (1~10점)
- 긴급도 (Urgency): 경쟁사 대비 열위 정도 (1~10점)
- 난이도 (Effort): 개발 복잡도 (1~10점, 역순)

총점 = (Impact + Urgency) / Effort

총점이 높은 순으로 우선순위를 정해줘.
```
