## ⏰ 소요 시간

## 📖 시나리오

이전 세션에서 사용자 인터뷰(정성 데이터)를 분석했습니다.
이제 앱 사용 통계(정량 데이터)를 추가로 분석하여, 두 데이터를 통합해 더 정확한 의사결정을 내려봅시다.

**목표:**
- 정성 데이터 + 정량 데이터 통합 분석 (트라이앵귤레이션)
- 데이터 기반 우선순위 결정
- 실행 가능한 액션 아이템 도출
- 예상 임팩트 정량화

## 📋 실습 재료

앞 세션 인터뷰 데이터에 다음 통계 데이터를 추가로 사용합니다:

```text
=== 앱 사용 통계 데이터 (최근 30일) ===

검색 기능 사용 현황:
- 전체 검색 수: 1,250,000회
- 자동완성 사용률: 78%
- 필터 사용률: 45%
- 평균 검색 시간: 3.8초
- 결과 페이지 이탈률: 35%

페이지별 체류 시간:
- 검색 결과 페이지: 평균 28초
- 숙소 상세 페이지: 평균 1분 42초
- 예약 페이지: 평균 3분 15초

클릭 히트맵 (모바일):
- 검색창: 95%
- 필터 버튼: 12% (찾기 어려움)
- 정렬 옵션: 8%
- 지도 보기 버튼 (없음): N/A

사용자 이탈 지점:
- 검색 결과 대기 중: 25%
- 필터 설정 중: 15%
- 숙소 상세 페이지: 10%
- 예약 페이지: 5%
```

### 데이터 트라이앵귤레이션 분석하기

Copilot Chat에 다음과 같이 요청하세요:

```
이전에 분석한 UX 인터뷰 데이터와 위의 통계 데이터를 결합하여 종합 분석해줘.

다음 내용 포함:
1. 정성/정량 데이터가 일치하는 부분 (트라이앵귤레이션)
   - 인터뷰에서 나온 불만이 실제 수치로도 확인되는 항목
   
2. 데이터 간 불일치 사항 및 추가 조사 필요 항목
   - 인터뷰에서 언급되지 않았지만 수치상 문제인 부분
   - 인터뷰에서 언급되었지만 수치상 큰 문제 아닌 부분

3. 데이터 기반 개선 우선순위
   - 정성+정량 데이터 모두에서 문제인 항목 (최우선)
   - 정량 데이터만 문제인 항목 (높음)
   - 정성 데이터만 문제인 항목 (중간)

4. 예상 개선 효과 (정량적 목표)
   - 각 개선사항이 적용되면 예상되는 수치 변화

표와 bullet points를 활용해서 보기 쉽게 정리해줘.
```

## 🎯 실습 2: 인사이트를 액션 아이템으로 변환

분석 결과를 실행 가능한 작업으로 변환해봅시다.

### 우선순위별 액션 아이템 만들기

```
위의 UX 리서치 종합 분석을 바탕으로 
실행 가능한 액션 아이템 리스트를 만들어줘.

각 액션 아이템마다 다음 정보 포함:
- 작업 내용 (구체적으로)
- 담당 팀 (UX/개발/마케팅/데이터 등)
- 예상 소요 기간 (일 단위)
- 예상 난이도 (상/중/하)
- 성공 측정 지표 (KPI)
- 기대 효과 (정량적)
- 우선순위 (P0/P1/P2)

표 형식으로 작성하고, 우선순위별로 그룹핑해줘.

추가로:
- 팀 간 의존성 표시 (A팀 완료 후 B팀 시작)
- 타임라인 (간트 차트 스타일)
```

### 도전 1: 경쟁사 비교 분석

### 도전 2: 사용자 여정 지도 생성

Copilot에게 다음과 같이 요청해보세요:

```
가장 불만이 많았던 "빠른 예약형" 페르소나의
사용자 여정 지도(User Journey Map)를 만들어줘.

각 단계마다:
- 터치포인트
- 사용자 행동
- 감정 (😊/😐/😞)
- 불만사항
- 개선 기회

Mermaid 다이어그램 또는 표 형식으로 작성해줘.
```
