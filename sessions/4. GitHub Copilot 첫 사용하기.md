# 4. GitHub Copilot 첫 사용하기

## ⏰ 소요 시간

**10분**

***

## 📖 시나리오

이제 환경 설정이 완료되었으니 GitHub Copilot을 직접 사용해봅시다! 마치 똑똑한 동료와 대화하듯이 Copilot과 소통하는 방법을 배웁니다.

**이 세션의 목표:**

* Copilot Chat 사용법 익히기
* 첫 프롬프트 작성 경험
* 효과적인 프롬프트 패턴 학습

***

## 💬 실습 1: 첫 대화 시작하기

### Step 1: Copilot Chat 열기

왼쪽 사이드바의 Chat 아이콘을 클릭해 Copilot Chat 창을 연 뒤 아래 프롬프트를 입력해봅니다.

### Step 2: 간단한 인사

**프롬프트**:

```
안녕하세요! 저는 디자이너 입니다. Copilot이 무엇을 도와줄 수 있나요?
```

**기대 결과**:

* Copilot이 자신의 기능을 설명합니다
* 디자이너가 활용할 수 있는 방법을 제안합니다

***

## 📊 실습 2: 간단한 문서 생성하기

### Step 1: 문서 생성 요청

**프롬프트**:

```
회의 참석자 5명의 이름과 역할을 표로 만들어줘. 
참석자는 김철수(PO), 이영희(UX), 박지민(마케터), 최민수(개발자), 정수진(디자이너)야.
```

**기대 결과**: Copilot이 다음과 같은 표를 생성합니다:

| 이름  | 역할   |
| --- | ---- |
| 김철수 | PO   |
| 이영희 | UX   |
| 박지민 | 마케터  |
| 최민수 | 개발자  |
| 정수진 | 디자이너 |

***

## ✍️ 실습 3: 좋은 프롬프트 작성법 익히기

### 핵심 원칙

**구체적으로 말하기**

```
❌ "보고서 만들어줘"
✅ "지난주 프로젝트 진행 상황 보고서를 만들어줘. 
    완료된 작업, 진행 중인 작업, 다음 주 계획으로 구성해줘."
```

**형식 지정하기**

```
❌ "데이터 정리해줘"
✅ "이 데이터를 표 형식으로 정리해줘."
```

**핵심**: 명확하게 말할수록 원하는 결과가 나옵니다.

***

## 🎯 예상 결과물

이 세션을 완료하면:

* ✅ Copilot Chat을 자유롭게 열고 사용
* ✅ 간단한 표와 문서를 생성할 수 있음
* ✅ 좋은 프롬프트 vs 나쁜 프롬프트를 구분
* ✅ 명확하고 구체적인 요청 작성
